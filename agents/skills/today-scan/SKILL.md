# 今日扫描

## 目标

一次扫描产生两类前端数据：

```txt
content/local/news/YYYY-MM-DD.json
content/local/job/eligible-jobs.json
```

并通过确定性脚本生成：

```txt
content/local/markdown/YYYY-MM-DD-daily-news.md
content/local/markdown/YYYY-MM-DD-job-scan.md
content/local/scan/YYYY-MM-DD.json
```

页面只消费文件。不要从页面发起联网抓取、模型调用或扫描任务。

## 开始前读取

1. 执行 `python scripts/ensure_profile.py`。若不存在 `data/profile.local.json`，让脚本通过终端问询生成本地画像；存在则直接继续。
2. 读取 `data/profile.local.json`，提取可验证的报考条件。
3. 读取 `data/job-sources.json`，把其中登记的每一个官方入口全部扫描并记录结果，不得只挑选部分渠道或只返回前几条岗位。
4. 读取 `agents/schema/daily-news.schema.json` 与 `agents/schema/eligible-jobs.schema.json`，保持输出结构。
5. 若公告出现画像未提供的硬性条件，例如户籍、党员身份、学位、资格证、工作年限，先向用户确认缺失信息，再判断是否属于符合岗位。

## 扫描一：每日时政

1. 以中国政府网、新华社/新华网、人民日报/人民网、国务院部门官网为主，检索扫描日期当天发布的内容。
2. 优先选择有公考积累价值的政策、法治、民生、治理、经济与公共安全事项。
3. 对每条纳入资讯的原文 URL 进行实际访问验证。只保留可打开的原文或官方转载页。
4. 保留标题、来源、发布时间、原文 URL、事件事实，并补充清晰标识为备考加工的申论角度、行测关联、积累表达与可能题目。
5. 写入 `content/local/news/YYYY-MM-DD.json`。不要用旧日内容冒充当日资讯；当天没有足够可靠条目时，写明实际条数。

## 扫描二：岗位

### 范围

以画像指定地区和 `data/job-sources.json` 为准，当前至少覆盖：

```txt
国家公务员、军队文职
北京、天津、雄安新区
石家庄市井陉县、鹿泉区、井陉矿区、藁城区、栾城区、正定县
公务员、事业单位、教师、医疗卫生、社区工作者、官方发布国有企业正式岗位
```

发现新的官方发布渠道时，核验后补充到 `data/job-sources.json`。排除中介转载、劳务派遣和无法回溯官方公告的条目。

### 符合岗位原则

1. 将能够确认符合画像硬性条件的岗位全部写入 `positions`，包括 `报名中`、`即将报名`、`已截止` 或 `已结束`；状态必须准确，便于历史对照。
   不得只输出推荐前几名、样例岗位或摘要数量；每个已确认符合条件的岗位都必须进入列表供前端筛选查看。
2. 不因为报名已截止而丢弃一个确实符合条件、对用户有参考价值的岗位。
3. 不因为希望页面有内容而放宽硬性资格；不能确认符合时，记录在扫描结论中，不放入 `positions`。
4. 每次扫描先核查国家公务员局当前年度及下一招录年度专题。若下一年度官方职位表已发布，必须下载附件并对目标地区、目标类别全量筛选；若未发布，写明检查日期与官方入口未发布结论，不得凭预测生成岗位。
5. 国企岗位只收官方正式招聘岗位；正式公告或单位官网公布待遇时记录原文。未公布金额但需要供用户比较时，可以按单位性质与地区给出宽区间推算，必须标记为“推算区间，非招聘单位待遇承诺”，不得引用无来源网帖。
6. 地方事业单位、国企和区县扫描结论写入 `regionalScanNote`；国考同步会合并该字段，不覆盖地方核验结果。

### 每条岗位需整理的信息

尽可能提取并展示：

```txt
单位、部门、岗位名称、岗位代码、工作地点、招录人数
工作内容（可在官方岗位说明基础上归纳日常职责，并注明推断）、学历/专业/届别/资格条件、报名时间、考试时间、状态
匹配度、推荐结论、匹配原因、风险等级、风险提醒与备考建议
公告备注和报考要点、住房与户口是否有官方说明
往年对应或高度可比岗位的进面/入围分、报录比、招录人数
工资/薪酬/福利的官方公开信息，或明确标为推算的宽区间参考
公告原文、往年数据来源、薪酬来源及采集时间
```

进面分、报录比不是所有单位都会公开，没有可追溯来源时显示“官方未公开”。薪资若没有官方金额，只能提供基于单位性质和地区的宽区间推算，且必须清晰声明非官方承诺。

## 收口与验收

先同步国家公务员局、石家庄市人社局和中国雄安官网的已接入官方附件，并检查下一年度国考入口；再核验和写入其他地方及国企结果。最终收口时执行：

```bash
python scripts/ensure_profile.py
python scripts/import_national_jobs.py
python scripts/today_scan.py --date YYYY-MM-DD
npm run typecheck
npm run build
```

`import_national_jobs.py` 同步国家公务员局、石家庄市人社局和中国雄安官网官方附件中能够依据画像确认的岗位，并记录已公开的官方考试数据；其他地方事业单位、国企与新增公告仍需 Skill 核验并写入数据。`today_scan.py` 负责校验 JSON Schema、生成 Markdown 与扫描清单。若任一纳入数据的信源未经实际验证，不应宣称扫描完成。

## 回报格式

向用户说明：

```txt
扫描日期
时政收录数量及权威来源
岗位官方入口覆盖数量
符合岗位数量、状态分布
因硬性条件排除或待确认的重点公告
数据文件与验证结果
```
