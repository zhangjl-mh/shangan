# 岗位扫描

## 输出

更新岗位判断工作台读取的数据：

```txt
content/local/job/eligible-jobs.json
content/local/markdown/YYYY-MM-DD-job-scan.md
content/local/scan/YYYY-MM-DD.json
```

## 开始前

1. 执行 `python scripts/ensure_profile.py`。若画像不存在，由脚本在终端问询并写入 `data/profile.local.json`。
2. 读取 `data/profile.local.json`、`data/job-sources.json` 与 `agents/schema/eligible-jobs.schema.json`。
3. 画像只用于本地资格筛选；公开输出不得写入姓名、毕业年份、画像哈希或可识别的原始个人字段。

## 三类范围

页面与数据只使用以下三种顶层类别：

```txt
公务员
编制
国企
```

教师或医疗招聘仅在属于正式编制招聘且符合画像时归入 `编制`。国企仅纳入官方发布的正式招聘，排除劳务派遣、外包和无法追溯用人单位或主管部门公告的转载。

## 必须按顺序执行

1. 检索 `data/job-sources.json` 登记的全部官方入口，并优先寻找最近发布、考试尚未举行的招录公告。
2. 对每个相关公告打开官方原文，核实公告发布日期、报名起止时间、考试时间、岗位附件 URL 和公告状态。
3. 下载官方岗位表或岗位附件，对全部行按画像中的地区、类别与硬性资格条件筛选，不得只抽样或仅返回前几名。
   若公告正文已经规定适用于所有岗位的硬性条件且画像明确不满足，例如定向身份或统一户籍限制，应记录公告级排除原因，可以不再下载附件。
4. 将筛选后仍处于 `报名中`、`即将报名` 或 `待考试`，且考试尚未举行的全部岗位写入 `positions`。
5. 针对每一个进入 `positions` 的岗位，继续检索公告正文或同一官方发布渠道，整理福利待遇、住房/租房支持、落户/户口说明及其来源。
6. 将实际检查的入口、下载附件结果、被排除的重点公告与原因写入 `searchedSources` 和 `screeningNote`。

## 时效红线

- 已笔试、已面试或已完成录用流程的岗位不得写入 `positions`，也不得在页面作为岗位卡片展示。
- 已报名截止但考试尚未举行的岗位可以写入 `positions`，状态标为 `待考试`，并在风险提醒中说明无法新报名；如果已经考试则只作为 `searchedSources` 中的扫描留痕。
- 每次检查国家公务员局当前及下一招录年度专题。新年度官方附件发布后必须下载并全量筛选；未发布时写明核验日期与未发布结论，不推测岗位。
- 不能确认硬性条件符合时，不纳入岗位列表；在核验记录中说明缺少的依据或排除原因。

## 页面重点字段

每个可展示岗位尽可能整理：

```txt
单位、岗位、类别、地区、状态、招录人数、岗位代码、公告 URL、采集时间
报名起止时间、资格初审、缴费截止、考试时间、工作内容或标明推断的职责归纳
matchScore、matchLevel、recommendation、riskLevel、riskReminders、studyAdvice
benefits、compensationReference、housingReference、householdReference
可追溯的最低进面分、报录比或官方未公开说明
```

学历、专业、届别和资格条件必须用于筛选并可保留在 JSON 中，但不是岗位页的主要展示信息。页面优先呈现：

```txt
福利待遇
房子 / 住房保障 / 租房支持
户口 / 落户条件
报名时间与风险
```

没有官方待遇、住房或落户说明时，直接写“官方公告未载明”，不要自行补成政策承诺。薪资金额未在公告公开时，仅可给出明确标为“推算区间，非招聘单位待遇承诺”的辅助范围。

## 收口

如当日时政 JSON 已存在，执行：

```bash
python scripts/import_national_jobs.py
python scripts/today_scan.py --date YYYY-MM-DD
npm run typecheck
npm run build
```

回报扫描日期、官方入口覆盖数、仍可展示岗位数量、重点排除公告及理由、数据与页面验证结果。
